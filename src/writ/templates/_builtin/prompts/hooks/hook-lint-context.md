## Type Context: Context Document

This file appears to be a **context document** (detected from its folder location).
Context documents describe project structure, domain knowledge, or environment.

Adapt your review with these context-specific priorities:

- **Currency**: Are file paths, version numbers, command examples, and
  statistics still accurate? Context docs rot fast when code changes.
- **Minimality**: Does the document repeat information an LLM already knows
  (language syntax, well-known library APIs)? Every redundant token is waste.
- **Structural accuracy**: If the document contains a treeview or file listing,
  does it match the actual directory structure?
- **Audience clarity**: Is this context for the developer, for an AI agent,
  or for an external reviewer? The level of detail should match the consumer.
- **No code duplication**: Context docs should describe architecture and
  conventions, not reproduce source code that will drift out of sync.
